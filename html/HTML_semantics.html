HTML 시맨틱

  1. 시맨틱 태그를 사용해야 하는 이유
    • Semantic, 시맨틱이란 의미가 부여 된다는 의미이다.
    • HTML 문서의 가독성과 유지보수가 쉬워진다.
    • 블럭 요소가 어느 영역에 있는지 예측하거나 확인해서 수정에 용이하다.
  2. 시맨틱 태그의 종류
    • <header>페이지의 상단에 포함되는 내용을 묶을 때 사용한다.</header>
    • <main>메인에 포함하는 모든 부분을 묶을 때 사용한다.</main>
    • <section>제목별로 나눌 수 있는 문서의 컨텐츠 영역을 구성할 때 사용한다.</section>
    • <article>개별 문서를 인용했거나, 작성한 글임을 표시할 때 사용한다.</article>
    • <aside>좌측이나 우측에 사이드바 또는 사이드에 있는 정보들을 묶을 때 사용한다.</aside>
    • <footer>페이지의 바닥부분, 페이지 정보 등을 묶을 때 사용한다.</footer>
    • <hgroup>h1, h2 등 제목과 부제목을 묶을 때 사용한다.</hgroup>
    • <nav>페이지의 메뉴를 묶을 때 사용한다.</nav>
